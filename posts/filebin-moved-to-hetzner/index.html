Filebin.net has moved to Hetzner

Published Tuesday, November 21, 2023 (500 words, 3 minutes to read).

About the move

Yesterday, on the 20th of November 2023, the hosting of the filebin.net service was moved from Redpill Linpro in Norway to Hetzner in Germany.

Some history

Filebin was a hobby project that let me learn the HTML5 File API. The end result made it easy to share photos and videos with friends and family, which was difficult at the time for non technical people.

The first version of filebin.net launched on the 27nd of July 2012. It was hosted by Redpill Linpro for free, and they continued to sponsor hosting and bandwidth for 11 years. This made it possible to provide the Filebin service all this time.

Since the beginning, Filebin has been rewritten a few times. The source code for the current version is written in Go and is available in Github. It can obviously be self-hosted, should you want to.

The design goal for Filebin has always been that it should be possible for non-technical people to share and receive files online. Even as I write this in 2023, it is still surprisingly difficult to share files online for non-technical people. The user interfaces are typically bloated, photos are often compressed and file sizes have to be limited.

Abuse

A service like this has the potential to be abused. Over the years, I’ve considered abuse mitigation features trying not to reduce ease of use for non-technical people. Some of the mitigations currently in place are:

  • Anyone that knows the URL to a file can delete it. This is supposed to make it harder to mass distribute files.
  • Limit the number of downloads for each file. This is supposed to make it harder to mass distribute files.
  • Reject certain file extensions. This is supposed to make it harder to rely on Filebin for phishing.
  • Some file extensions are downloaded as attachments and not rendered in the browser. This is supposed to make it harder to use Filebin for phishing.
  • Rate limits here and there.
  • IP addresses are logged and shared with law enforcement by request.
  • IP addresses that are found to upload malicious content are blocked.

All nice things eventually come to and end, and the service will have to shut down the day abuse mitigation work takes too much time.

Ads

The costs associated with the service have been limited to the domain name up until now, which has made it possible to provide the service without ads. The new hosting environment at Hetzner comes with a cost, and I consider introducing a banner ad to offset some of this cost. I’d prefer not to, but I prefer less to carry the entire cost.

Hosting environment

Up until now Filebin has been running on a small virtual machine with 2 vCPUs and 4 GB of memory. The storage capacity has been limited to 200 GB. As of yesterday, the service is running on a dedicated physical server with 64 GB of memory and 3 TB storage capacity.
